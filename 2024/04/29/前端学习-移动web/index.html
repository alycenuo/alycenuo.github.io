1、平面转换 transform

作用： 又称2D转换，为元素添加动态效果，一般与过渡配合使用
概念：改变盒子在平面内的形态（位移、旋转、缩放、倾斜）

  • 一般配合transition（过渡属性使用）
  • vscode简写为tf

平面转换-平移

属性： transform： translate(X轴移动距离,Y轴移动距离);

  • 取值
    写法一： 像素单位数值
    写法二： 百分比（参照盒子自身尺寸计算结果）
    正负均可

  • 技巧
    translate()只写一个值，表示沿着X轴移动
    单独设置X或Y轴移动距离： translateX()或translateY()

定位居中

双开门效果

平面旋转-旋转

属性： transform: rotate(旋转角度);

  • 角度单位是deg
  • 取值为正，顺时针旋转；取值为负，逆时针旋转

改变转换原点

默认情况下，转换原点是盒子中心点
属性： transform-origin: 水平原点位置 垂直原点位置;

取值写法：

  1. 方位名词（left、top、right、bottom、center）
  2. 像素单位数值
  3. 百分比

多重转换

transform: translate() rotate()

缩放

属性： transform: scale(缩放倍数);
属性： transform: scale(X轴缩放倍数,Y轴缩放倍数);

  • 只设一个值表示等比缩放
  • 大于1表示放大，小于1表示缩小

倾斜效果

属性： transform: skew();
取值： 角度度数deg

属性：

  • 渐变方向写法（可选）
  1. to 方位名词
  2. 角度度数
  • 终点位置写法（可选）
  1. 百分比

渐变案例

3、径向渐变

作用： 一般用于给按钮添加高光效果

属性

取值

  • 半径可以是2条，此时为椭圆
  • 圆心位置取值： 像素/百分比/方位名词

4、空间转换

  • 又称3D转换，Z轴方向与视线方向相同
  • 属性： transform

取值

  • 像素单位数值
  • 百分比（参照盒子自身尺寸）

5、视距 perspective

作用： 指定观察者与z=0平面的距离，为元素添加透视效果
透视效果： 近大远小、近实远虚

属性： perspective: 视距;（添加给直接父级，取值范围一般为800-1200）

6、空间转换-旋转

绕Z轴旋转： transform: rotateZ（360deg）;

  • 效果与平面旋转一致

绕X轴旋转： transform: rotateX(60deg);
绕Y轴旋转： transform: rotateY(60deg);

  • 添加视距属性可以使效果更明显

左手法则

  • 根据旋转方向确定取值正负
  • 左手握住旋转轴，拇指指向正值方向，其它四个手指弯曲方向为旋转正值方向

扩展-自定义旋转轴位置

  • rotate3d(x,y,z,角度度数)
  • 自定义旋转轴的位置及旋转角度
  • x,y,z取0-1之间的数字

  2. 子级 定位

9、动画 animation

使用

区别

  • 过渡： 实现 两个状态间的变化过程
  • 动画： 实现 多个状态间的变化过程，动画过程可控（重复播放、最终画面、是否暂停）

实现步骤

1.定义动画

2.使用动画

animation: 动画名称 动画花费时长;

属性值

animation: 动画名称 动画时长 速度曲线 延迟时间 重复次数 动画方向 执行完毕时状态;

  • 动画名称和动画时长必须赋值
  • 取值不分先后顺序
  • 如果有 两个时间值，第一个时间表示 动画时长，第二个表示 延迟时间

animation拆分写法

  • 名字很长，一般比较少用
属性 作用 取值
animation-name 动画名称
animation-duration 动画时长
animation-delay 延迟时间
animation-fill-mode 执行完毕状态 forward、backward
animation-timing-function 速度曲线 steps(数字)；逐帧动画
animation-iteration-count 重复次数 infinite为无限循环
animation-direction 动画执行方向 alternate为反方向
animation-play-state 暂停动画 paused为暂停，通常配合 :hover使用

10、屏幕分辨率、视口标签、二倍图使用

屏幕分辨率

水平和垂直方向上的像素点数，单位是px，可分为物理分辨率和逻辑分辨率，网页匹配的是逻辑分辨率

  • 硬件分辨率： 物理分辨率（出厂设置）
  • 缩放调节的分辨率： 逻辑分辨率（软件/驱动设置）

视口

显示HTML网页的区域，用来约束HTML尺寸

  • width=device-width： 视口宽度 = 设备宽度
  • initial-scale=1.0： 缩放1倍（不缩放）

二倍图使用

概念： 设计稿里面每个元素的尺寸的倍数
作用： 防止图片在高分辨率屏幕下模糊失真

iPhone6/7/8 物理分辨率750 x 1334 逻辑分辨率375 x 667

在当今年代，iPhone6/7/8产出设计稿，如果宽为750px一般为二倍图，像素大厨或HTML里面都应选择 2x模式使用

12、rem的基本使用

  • rem单位是 相对单位
  • rem单位是相对于 HTML标签的字号的计算结果
  • 1rem = 1HTML字号大小

13、媒体查询

  • 媒体查询能够 检测视口的宽度，然后编写 差异化的CSS样式
  • 当某个 条件成立，执行对应的CSS样式

媒体查询配合rem

目前rem布局方案中，将网页等分为 10份，HTML标签的字号为 视口宽度的 1/10

  • 实际工作中会搭配JavaScript使用，并不需要每个视口单独设置媒体查询

rem-flexible.js

  • flexible.js是一个用来 适配移动端的 js库
  • 核心原理就是根据不同的视口宽度给网页中 HTML根节点设置不同的font-size

14、rem-移动适配

设计稿中的单位是px而非rem，因此需要将px转化为rem

  1. 确定基准根字号，一般是所选视口宽度的1/10
  2. rem单位的尺寸 = px单位数值/基准根字号

15、less

  • less是一个 CSS预处理器，Less文件后缀是 .less。扩充了CSS语言，使CSS具备一定的逻辑性、计算能力

  • 目前浏览器不识别less代码，需要转为对应的CSS文件再引入

  • VS Code插件： Easy LESS，保存less文件后自动生成对应的CSS文件

less注释

单行注释： 不可换行
语法： // 注释内容
快捷键： ctrl + /

块注释： 可换行
语法： /* 注释内容 */
快捷键： Shift Alt A

  • css文件并不支持单行注释，less文件转为css文件后，less单行注释会消失，多行注释不变

less运算

运算： 加减乘直接写，除法需要加/或.

less嵌套

作用： 快速生成后代选择器

less-变量

概念： 容器，存储数据
作用： 存储数据，方便使用和修改

语法：

  1. 定义变量： @变量名:数据;
  2. 使用变量： CSS属性:@变量名;

less导入-导出-禁止导出

导入
作用： 导入less公共样式文件
语法： 导入：@import "文件路径";
提示： 如果是less文件可以省略后缀

导出
语法： 在less文件的第一行添加// out:存储URL

禁止导出
写法： 在less文件第一行添加// out:false
提示： 保存后不再自动生成css文件

16、vw适配方案

  • 相对单位
  • 相对视口的尺寸计算结果
  • vw： viewpoint width 1vw = 1/100视口宽度
  • vh： viewpoint height 1vh = 1/100视口高度
  • 开发中不能将vw和vh混用，全面屏视口高度尺寸大，可能导致盒子变形

21、Bootstrap框架

中文官网： https://www.bootcss.com/

  • 按照框架的规则去使用它的功能
  • 下载： 想要查看其实现方式可以下载源码，使用功能则下载生产文件
  • 解压后的文件中，带mini后缀的是压缩后的文件，代码多放在一行，人很难解读，功能相同，文件体积小，如果只是要用其功能而不进行修改，可以用这个

使用：

  1. 引入bootstrap的css文件
  2. 调用类名： container: 响应式布局版心

栅格系统

栅格化是将这个网页宽度分成12等份，每个盒子占用对应份数，例如一行排4盒子，则每个盒子占3份即可（12/4=3）

常用布局类

  • col--： 列（例如： col-xxl-3）
  • row： 行

响应断点划分区间，类前缀标上类的占用份数

xs sm md lg xl xxl
<576px >=576px >=768px >=992px >=1200px >=1400px
container(max-width) None(auto) 540px 720px 960px 1140px 1320px
.col- .col-sm- .col-md- .col-lg- .col-xl- .col-xxl-

全局样式、组件

  • 全局样式
    例如button类、table类，具体效果和用法可查看bootstrap框架使用文档

  • 组件（components）
    使用组件可以快速完成对应功能，提高开发效率

  1. 引入样式表（框架的官方文档的components项）
  2. 引入js文件（如果无动态则无需引入）
  3. 复制结构，修改内容

如果要修改效果，可以通过F12模式查看源代码对应的类名，然后重新写样式层叠掉框架的效果

字体图标

  • 下载
    导航/Extend： 图标库 → 安装 → 下载安装包 → bootstrap-icons-1.X.X.zip

  • 使用

  1. 复制 font文件夹到项目目录
  2. 网页引入 bootstrap-icons-css文件
  3. 调用 CSS类名（图标对应的类名）
